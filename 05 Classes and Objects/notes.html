Classes and Objects

Classes
  • Define a user defined type
  • Should derive from object - new style classes
  • Older classes do not derive from object - old style classes
  • Supports full Multiple Inheritance
  • Several design flaws - use decorators to overcome
Key Points
  • Classes are similar to classes in Java, C# and C++, but ...
    • No data encapsulation - no public and private
    • Constructors - can only have one
    • self - instead of this
  • Attributes and Methods - are objects themselves
    • Object's dictionary - stores attributes
    • Class's dictionary - stores methods
  • statics - implemented as class attributess
